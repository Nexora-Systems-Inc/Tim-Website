# Progress

Periodic project status reports during active development.

Use this folder to maintain a running record of where the project stands at agreed checkpoints — weekly updates, sprint summaries, milestone reviews, or pre-launch status reports.

## Contents

| Folder / File | Purpose |
|---------------|---------|
| [`reports/`](reports/) | Individual progress report documents |

## Report Naming Convention

```
reports/YYYY-MM-DD-status.md
reports/YYYY-MM-DD-milestone-[name].md
```

Example: `reports/2026-03-15-status.md`

## Suggested Report Sections

Each report in `reports/` should typically include:

- **Reporting period** — dates covered
- **Summary** — one-paragraph executive overview
- **Completed** — work finished since last report
- **In progress** — active work items
- **Blockers / risks** — items needing attention
- **Next steps** — planned work before next report
- **Client actions needed** — outstanding items from the client
