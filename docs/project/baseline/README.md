# Baseline

Original client requirements and project scope definition.

This folder establishes the **source of truth** for what was agreed before and during project kickoff. All downstream work — progress reports, meeting notes, and completion documentation — should trace back to items defined here.

## Contents

| File / Folder | Purpose |
|---------------|---------|
| [`baseline.md`](baseline.md) | Master baseline document: requirements, email log, decisions, and references |
| [`revisions.md`](revisions.md) | Change log when the baseline is updated after initial sign-off |
| [`appendix/`](appendix/) | Supporting reference material such as screenshots, client markups, photographs, PDFs, and attachments |

## Workflow

1. Copy or adapt `baseline.md` at project start.
2. Log client emails and key communications as numbered entries (`E-001`, `E-002`, …).
3. Store attachments in `appendix/` and link to them from the baseline.
4. Record any post-sign-off changes in `revisions.md`.
